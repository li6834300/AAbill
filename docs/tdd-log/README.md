# TDD 日志

每个功能一篇,命名 `NNN-功能名.md`。这里记录的不是代码(git 历史里有),而是**过程**:人给了什么规格、AI 先写了什么测试、人否决/修正了什么。

## 模板

```markdown
# NNN: <功能名>

日期: · 相关 commits: <test/feat/refactor 三个 hash>

## 验收标准(人写)
Given / When / Then …

## 红:AI 写的测试
- 覆盖了哪些场景,人工补充/删掉了哪些
- 审查中发现的问题(断言太松?漏边界?)

## 绿:实现
- AI 首版实现被否决或修正的点

## 重构
- 做了什么,为什么

## 学到的
一两句。
```
